Issue #91 — Tailwind 4 Text Misalignment

Root cause analysis for panphora/overtype#91

Root Cause: CSS Strut + Font Mismatch on <pre>
Tailwind 4's preflight sets pre { font-family: ui-monospace, ... }. Overtype hardens font-family on <code> but not on <pre> itself. Each line box inside <pre> has an invisible "strut" using <pre>'s font metrics. When the strut's font has different ascent/descent than <code>'s font, line boxes inflate by a few pixels per line — shifting all subsequent overlay text out of alignment with the textarea cursor. Computed line-height reports identically for both, making the bug invisible in DevTools.
Fix: Add font-family: inherit !important to .overtype-wrapper .overtype-preview pre in src/styles.js, so the <pre> inherits the same font as the textarea.
